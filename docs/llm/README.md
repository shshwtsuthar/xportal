# XPortal Student Management System - Comprehensive Documentation

## Overview

This directory contains extensive documentation for the XPortal Student Management System (SMS), a comprehensive platform designed for Australian Registered Training Organisations (RTOs). The documentation covers all aspects of the system including architecture, development pipelines, implementation progress, and operational procedures.

## Key Technologies

### Backend
- **Supabase** - Backend-as-a-Service with PostgreSQL database
- **Deno** - Runtime for Supabase Edge Functions
- **TypeScript** - Type-safe development
- **OpenAPI 3.0** - API specification and documentation

### Frontend
- **Next.js 14** - React framework with App Router
- **TypeScript** - Type-safe development
- **Tailwind CSS** - Utility-first styling
- **ShadCN UI** - Component library
- **React Hook Form** - Form management
- **Zod** - Runtime validation
- **Zustand** - State management

### Development Tools
- **Docker** - Containerization for Supabase
- **Postman** - API testing and documentation
- **ESLint & Prettier** - Code quality and formatting

## Compliance & Standards

### Australian RTO Compliance
- **AVETMISS 8.0** - Australian Vocational Education and Training Management Information Statistical Standard
- **CRICOS** - Commonwealth Register of Institutions and Courses for Overseas Students
- **ESOS Act** - Education Services for Overseas Students Act
- **ASQA** - Australian Skills Quality Authority standards

### Technical Standards
- **WCAG 2.1 AA** - Web Content Accessibility Guidelines
